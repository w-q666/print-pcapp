# 打印服务发现与系统配置 — 记忆摘要

> 用途：冷启动局域网 Java 打印服务发现、系统配置项、异常跳转等行为速查。下次提问可直接引用本文件路径。

## 一句话

桌面端启动时用 Rust 并发探测 Java 打印服务（`GET /print/getPrintServers`，`code==0`）；失败则试默认地址；仍失败则**自动进入「系统配置」**；配置集中在页面顶部「打印服务」区块。

## 行为记忆点

1. **扫描范围留空**  
   Rust 用 `if-addrs` 枚举本机 **非回环、非链路本地** IPv4，对每个所在 `/24` 展开 `*.1`～`*.254`，**去重**，**最多 300 个地址**，再探测。避免「主 IP 在 192.168.1.x、Java 在 192.168.137.x」永远扫不到的问题（在网卡枚举得到 137 段时）。

2. **用户填写起止 IP**  
   严格按该范围扫；须同 /24、连续、≤300；与 `docs/superpowers/specs/2026-05-15-lan-print-service-discovery-design.md` 一致。

3. **默认服务 IP**  
   始终**第一个**探测；可填 `localhost` 或 IPv4。服务在别机且本机无该网卡时，应直接填主机 IP 或收窄/写对网段。

4. **端口**  
   `servicePort` 可配置（默认 2024），与发现、`setBaseURL`、`wsUrl`、Rust `config.port` 同源。

5. **`scanRangeInferredOnce`**  
   任意一次在「系统配置」点「保存配置」会置 `true`；用于避免反复覆盖用户意图。与「多网卡留空自动扫」配合理解。

6. **冷启动 UI**  
   全窗半透明遮罩 + `Spin` + 时间型 `Progress`（约 10s 预算），`Teleport` 到 `body`，高 `z-index`。

7. **失败后跳转**  
   仅当：**未发现 `foundHost`** 且 **`getPrintServers()` 对当前默认地址仍失败** → `router.push({ name: 'settings' })`。若扫描没中但默认地址能连，**不跳转**。

8. **接口文档**  
   `docs/接口文档/HTTP接口文档.md` — 探测路径 `/print/getPrintServers`。

## 排障口诀

- 浏览器能访问 `http://IP:2024` 但应用找不到 → 看**扫描范围是否含该网段**、**默认 IP 是否写成该主机**、**端口是否一致**。  
- 配置页找不到项 → 在 **系统配置页顶部**（非「文件格式」子 Tab）。  
- 自动化 `cargo test discovery::tests` **不测真实 HTTP**，只测 IP 展开逻辑。
